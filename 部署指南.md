## 部署步骤

### 第一步：注册 Supabase 账号

1. 访问 https://supabase.com 并注册账号
2. 创建新项目
3. 获取项目 URL 和 Anon Key:
   - 进入 Project Settings → API
   - 复制 `Project URL` 和 `anon public` 密钥

### 第二步：配置数据库

1. 在 Supabase 项目中，进入 SQL Editor
2. 新建查询，复制并执行 `数据库初始化脚本.sql` 中的内容
3. 这将创建所有必需的表和安全策略

### 第三步：配置环境变量

1. 打开 `my-app/.env.local` 文件
2. 替换以下内容：
```env
NEXT_PUBLIC_SUPABASE_URL=https://你的项目ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的anon密钥
```

### 第四步：安装依赖

在项目目录下运行：

```bash
cd my-app
npm install
```

### 第五步：本地测试

```bash
npm run dev
```

访问 http://localhost:3000 查看效果

### 第六步：部署到 Vercel

#### 方法A：通过 Vercel CLI

1. 安装 Vercel CLI:
```bash
npm i -g vercel
```

2. 登录 Vercel:
```bash
vercel login
```

3. 部署:
```bash
vercel
```

#### 方法B：通过 GitHub + Vercel（推荐）

1. 创建 GitHub 仓库
2. 推送代码到 GitHub:
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

3. 在 Vercel 导入项目:
   - 访问 https://vercel.com/new
   - 导入 GitHub 仓库
   - 配置环境变量（同第三步）
   - 点击 Deploy

### 第七步：配置环境变量到 Vercel

1. 在 Vercel 项目设置中，进入 Environment Variables
2. 添加以下变量：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. 重新部署项目

## 使用说明

### 首次使用

1. 访问部署后的网站
2. 点击"注册账号"创建新账户
3. 查看邮箱，点击验证链接激活账户
4. 登录系统

### 上传考勤文件

1. 登录后进入仪表板
2. 点击上传区域或拖拽 Excel 文件
3. 系统自动解析并显示表格

### 编辑数据

1. 点击任意单元格进行编辑
2. 按 Enter 保存，Esc 取消
3. 使用工具栏添加行列
4. 使用搜索框过滤数据
5. 点击"导出"下载修改后的文件

## 常见问题

### Q: 如何修改数据库表结构？
A: 在 Supabase SQL Editor 中执行 ALTER TABLE 语句

### Q: 如何添加更多用户字段？
A: 修改 `profiles` 表结构，并更新相应代码

### Q: 如何配置邮件服务？
A: 在 Supabase Authentication → Email Templates 中配置

### Q: 如何查看日志？
A: 在 Vercel 项目页面查看 Deployment 日志
